Informs users about important changes or conditions in the interface. Use this
component if you need to capture the user's attention in a prominent way.

## Colors

Use the `color` attribute to set a theme color for the alert, each serving its own semantic purpose. These colors include: primary (default), secondary, danger, warning, success, and info.

## Accessibility

This component aims to meet the WCAG 2.1 AA accessibility guidelines.

- Danger and warning alerts have a `role="alert"` definition that makes sure assistive technologies announce them when they appear.
- Default, secondary, info and success alert variations have a `role="status"` definition. This makes sure they're being read after danger and warning alerts.
- Alerts additionally use `tabindex="0"` to make it possible to access them using the tab key or programmatically move focus to them.
